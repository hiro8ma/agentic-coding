# 週報の書き方ガイドライン

## トーン

- 敬語（です・ます調）で統一する
- 主語と動詞を明確にし、誰が何をしたかを残す
- 主観的形容詞より客観的な事実・数値を優先する

## 長さ

- 全体で 1 画面（おおむね 20 行以内）に収める
- 1 セクション 3〜7 項目を目安にする
- 1 項目 1 行。複数の話題を 1 行に詰めない

## セクションごとの観点

| セクション | 書くこと | 書かないこと |
|---|---|---|
| 実績 | 完了した成果と状態 | 着手しただけの作業を「完了」と書く |
| 課題 | ブロッカー・遅延・依存 | 解決済みの問題を未解決として残す |
| 次週予定 | 着手・完了予定と優先度 | 願望ベースの曖昧な計画 |

## NG 例と改善例

| NG | 改善 |
|---|---|
| いろいろ対応しました | 認証 API の改修を完了し、レビュー待ちです |
| だいたい終わりました | 5 件中 4 件を完了、残り 1 件は来週です |
| 問題が多いです | テスト環境の DB 接続が不安定で、再現調査中です |
| 来週も頑張ります | 来週は決済フローの結合テストを完了させます |

## 数値で書く

- 「ほぼ完了」→「90%（残タスク: ドキュメント更新）」
- 「少し遅れている」→「予定比 2 日遅れ」
