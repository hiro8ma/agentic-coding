# 週報作成スキル

週次の進捗報告を、読み手が数十秒で把握できる定型フォーマットで作成する。

## 手順

### 1. 情報収集

- 対象期間（今週）の完了タスク、進行中タスク、ブロッカーを集める
- コミット履歴・Issue・チケットがあれば事実ベースで拾う
- 推測で埋めず、不明点はユーザーに確認する

### 2. 構成

`assets/template.md` を雛形として使い、以下のセクションを埋める。

- 実績（今週やったこと）
- 課題・ブロッカー
- 次週予定

各セクションは箇条書き。1 項目 1 行を原則とする。

### 3. 執筆ルール

- 敬語で書く（です・ます調）
- 箇条書きを基本とし、1 文を短く切る
- 曖昧語（「ほぼ」「だいたい」）を避け、数値・状態で書く
- 詳細な書き方・トーン・NG 例は `references/guidelines.md` を参照する

### 4. 検証

完成したテキストをファイルに保存し、検証スクリプトで必須セクションの有無を確認する。

```bash
python skills/weekly-report/scripts/validate.py <report-file>
```

スクリプトが不足セクションを報告した場合は、該当セクションを追記してから再検証する。
