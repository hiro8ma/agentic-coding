# Slack Post

Slack 投稿文を作るときの規範。
プラットフォーム制約（後から直せない・流し読みされる）を前提に、投稿前に完成形へ整える。

## プラットフォーム制約（前提知識）

- **bot / API 経由の投稿は、投稿後に人間が編集・削除できない**。投稿前にフォーマットを完成させる。「あとで直す」は存在しない
- Slack の見出し記法はない。`*太字*` をセクション見出しとして使う
- 長文は自動で折りたたまれる。重要な結論は冒頭 3 行以内に置く
- コードやログは必ずコードブロックに入れる（インライン崩れ防止）

## フォーマット規則

- セクション見出し（`*1. xxx*` など）の**前後に空行**を入れる
- 段落ごとに改行を挟み、詰まった印象にしない
- 箇条書きは 1 項目 1 行。ネストは 1 段まで
- 絵文字は文頭の識別用に最小限（乱用しない）

## 文章規則（ローコンテキスト）

- 一文 15 語未満を目安に短く切る
- 曖昧語（「ほぼ」「おおむね」「少し」）は具体的な数字に置き換える
- 略語・専門用語は初見の人に通じる言葉を選ぶ。避けられない場合は括弧で展開
- 主語・動詞・目的語を明確にする（誰が / 何のために / 何をする）
- 能動態を使う。受動態を避ける
- 書いた文に「So What?」と自問し、答えがない文は削る
- 受信者の推測に頼らず、額面通りに読める文章にする

## 出力手順

1. 目的（報告 / 依頼 / アナウンス / 質問）と受信者を確認する
2. 結論を最初の段落に書く
3. 上記規則で整形する
4. **投稿見本をそのまま出力し、改行・見た目込みでユーザーに確認してもらう**（承認前に投稿しない）

## 型（依頼の例）

```
*お願い: ○○の確認（締切: 7/15 12:00）*

△△のため、○○の確認をお願いします。

*背景*

（1〜2 文で。リンクがあれば貼る）

*お願いしたいこと*

1. □□を確認する
2. 問題があれば本スレッドに返信する
```
