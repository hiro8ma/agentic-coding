# ナレッジ note の書き方ルール

## 要点主義

- 全文の引き写しをしない。主張・根拠・結論だけを残す
- 1 項目 1 トピック。複数概念を 1 行に詰めない
- 概念は必ず具体例とセットにする（例のない抽象語は読み手に伝わらない）

## 表を活用する

- 比較・分類・対応関係は文章より表にする
- 「A と B の違い」「N 個の選択肢」は表で並べると数秒で読める

## 中立な出典表記

- 出典は事実のみ書く（タイトル・公開日・媒体・URL）
- 出典への評価・感想（「名著」「必読」等）を出典行に混ぜない
- 出典は本文末尾に 1 箇所だけまとめる。本文中に散らさない

## 主観的形容詞を避ける

| 避ける | 置き換え |
|---|---|
| すごく速い | スループットが約 2 倍 |
| 軽量 | 依存ライブラリ 0、起動 50ms |
| ほぼ完璧 | 既知の制約は X のみ |

## 面接で語れる粒度

- 「何をしているか」を自分の言葉で説明できる状態にする
- なぜその設計が選ばれたか（トレードオフ）を 1 行で添える
- 自分の実装との接続を必ず書く。借り物の知識で終わらせない

## フォーマット規約

- フロントマターに title / date / tags は必須
- ファイル名は小文字・ハイフン区切り、章番号プレフィックスなし
- 日本語の文末に句点を多用せず、見出しと箇条書きで構造を見せる
