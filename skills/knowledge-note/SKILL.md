# 学習ナレッジ化スキル

学習教材を、後から面接や実装で再利用できる構造化ノートに落とす。単なる要約ではなく「人に説明できる粒度」で書く。

## 手順

### 1. 内容の把握

- 教材の主張・前提・結論を読み取る
- 専門用語は「何をしているか」を具体例で言い換えられる状態にする
- 出典情報（URL・タイトル・公開日）を控える

### 2. ファイル作成

- `assets/template.md` を雛形として使う
- ファイル名は `{topic-slug}.md`（小文字・ハイフン区切り）
- 章番号プレフィックスはつけない

### 3. 各セクションを埋める

- **フロントマター**: title / date / tags（必須）、出典があれば source
- **TL;DR**: 「何が」「なぜ」「どうなる」を 3 文以内
- **要点**: 箇条書きと表で。概念は具体例とセットにする
- **自分の実装との接続**: 手元のコード・プロジェクトにどう当てはまるかを 1〜2 段落
- **出典**: 中立な表記で 1 つにまとめる

### 4. スタイル適用

書き方ルール（要点主義・表の活用・中立な出典表記・主観的形容詞の排除）は `references/style.md` に従う。

### 5. 検証

- フロントマターに title / date / tags が揃っているか
- TL;DR が 3 文以内か
- 要点が「具体例つき」で書かれているか
- 出典が 1 箇所にまとまっているか
