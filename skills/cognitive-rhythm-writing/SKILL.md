# 認知リズムライティング

出典 [k16shikano 氏の gist「認知リズムを生むための日本語ライティング規範」](https://gist.github.com/k16shikano/eb2929f13ed19c97188393d297be8432)。
原文にライセンス表記がないため、本ファイルは原文の複製ではなく規範の要旨を自分の言葉で再構成したもの。適用に迷ったら原文を参照する。

japanese-tech-writing が「正確で誤解のない文」を作る規範であるのに対し、本規範は「読者が読み続ける文章」を作る規範。両方を併用する。

## 基本原理

- 緩急は情報量の増減ではなく、読者の認知モード（観察 → 迷い → 確信 → 再観察）の切り替えとして設計する
- 常に少なくとも 1 つ、未回収の緊張（まだ答えの出ていない問い、あとで返すと約束した答え）を開いておく。緊張がすべて閉じた瞬間、読者は読むのをやめられる
- 装置の名前を本文に書かない。実現するが宣言しない（「ここで問いを立てておく」のような実況は禁止)

## 実装の要点

**文の拍** — 短文で立てる → 長めの文で流す → 短文で止める。断定と逡巡を交互に置き、逡巡は後の展開への布石として使う。

**段落の密度** — 密な段落 2〜3 に対して疎の段落 1 つ。具体（数値・記録・発言）と意味づけを交互に置く。箇条書きは圧縮だけでなく「間」として使う。

**冒頭と節の入り** — 最初の数文で未回収の緊張を作る。「本節では〜を扱う」型の宣言は禁止。直前の違和感を問いで言い直す、読者の反問をそのまま書く、のいずれかで入る。理論・概念は違和感を先に見せてから命名として導入する（理論 → 例の順は禁止）。

**列挙の着地** — 性質や分類を列挙したら、各項目を直前の具体的場面へ 1 つずつ着地させる。

**問いの回収** — 立てた問いは明示的に回収する。分割回収（答えの半分を先に返し、残りを後段へ）で推進力を作れる。結びは抽象論のまま終えず、読者が既に持つ具体へ着地させる。

## 駄文の判定 — 話題テスト

判定軸は 1 つ。**その文は「状況」を更新しているか、「文書」を更新しているか**。

- 状況更新（出来事・データ・発言・書き手の判断状態の新情報）→ 残せる
- 文書更新（この章がどう見えるか、次に何を書くか、本文自身への言及）→ 原則削除。短くてリズムが良くても残す理由にならない

削除対象の典型 — 「ここまでだと概念説明に見えるだろう。なので例へ戻す」「要するに主題は〜ではなく〜」「次は〜を見ていく」。
残せる緩みの典型 — 「うまくいっているに違いない」（後で崩される思い込み）「今すぐでなくても整理は要る」（判断の保留)。どちらも状況・判断状態の更新である。

## 点検手順（推敲時に機械的に実行する）

1. **話題テスト** — 段落頭の文と独立短文をすべて拾い、状況か文書かを判定。文書更新は削除か書き換え。推敲で新しく足した文は駄文の最大の混入経路なので、直後に再テスト
2. **漏出テスト** — 「緊張」「回収」「答えの半分」など装置の語彙が本文に漏れていないか検索。あれば削除し、内容側で実現し直す
3. **緊張台帳** — 立てた問い・思い込み・約束を列挙し、それぞれの回収位置を指す。指せないものは回収を書くか、問いごと削除
4. **拍の点検** — 長い断定文が 3 つ以上連続する箇所に、短い足場・停止・逡巡を挟む
5. **境界の点検** — 二人称の呼びかけ・依頼は本文中盤に置かず、冒頭・結びの境界へ移すか削除
